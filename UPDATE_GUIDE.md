# FT Dashboard — Daily Update Guide

## Overview
The dashboard is generated by `gen_dashboard.py` → `index.html`.
All data is hardcoded as Python arrays in `gen_dashboard.py`.
The daily update process re-runs ClickHouse queries and updates those arrays.

## Database
All tables are in the `alright` ClickHouse database.
- `alright.chargeattempts` — columns: `_id`, `premiumPlanHistoryId`, `amount`, `chargeAttemptStatus`, `failureReason`, `createdAt`
  - `chargeAttemptStatus` values: `Completed`, `Failed`, `Abandoned`, `Initiated`
- `alright.premiumplanhistories` — columns: `_id`, `userId`, `isFreeTrial`, `createdAt`

## Critical: DISTINCT fix
All queries use `SELECT DISTINCT paid._id AS plan_id` in the `ft_paid` CTE to prevent
join fan-out (users with multiple FT records inflate charge attempt counts).

## Base CTEs (used in every query)
```sql
WITH ft_paid AS (
    SELECT DISTINCT paid._id AS plan_id, paid.userId
    FROM alright.premiumplanhistories paid
    INNER JOIN alright.premiumplanhistories ft
        ON paid.userId = ft.userId
        AND ft.isFreeTrial = 1
        AND ft.createdAt >= '2026-02-01'
    WHERE paid.isFreeTrial = 0
),
ranked AS (
    SELECT fp.plan_id AS plan_id,
           ca.amount AS amount,
           ca.chargeAttemptStatus AS status,
           ca.failureReason AS failureReason,
           ca.createdAt AS createdAt,
           ROW_NUMBER() OVER (PARTITION BY fp.plan_id ORDER BY ca.createdAt) AS att_num
    FROM alright.chargeattempts ca
    INNER JOIN ft_paid fp ON ca.premiumPlanHistoryId = fp.plan_id
    WHERE ca.chargeAttemptStatus != 'Initiated'
)
```

## Query 1 — Daily Revenue by Amount (Panel 1 chart)
Returns: charge_date, amount, completions, revenue
Used to update: `daily_raw` list in gen_dashboard.py

```sql
WITH ft_paid AS (
    SELECT DISTINCT paid._id AS plan_id
    FROM alright.premiumplanhistories paid
    INNER JOIN alright.premiumplanhistories ft
        ON paid.userId = ft.userId AND ft.isFreeTrial = 1 AND ft.createdAt >= '2026-02-01'
    WHERE paid.isFreeTrial = 0
)
SELECT toDate(ca.createdAt) AS charge_date,
       ca.amount,
       countIf(ca.chargeAttemptStatus = 'Completed') AS completions,
       sumIf(ca.amount, ca.chargeAttemptStatus = 'Completed') AS revenue
FROM alright.chargeattempts ca
INNER JOIN ft_paid fp ON ca.premiumPlanHistoryId = fp.plan_id
WHERE ca.chargeAttemptStatus != 'Initiated'
  AND toDate(ca.createdAt) >= '2026-03-01'
GROUP BY charge_date, ca.amount
HAVING completions > 0
ORDER BY charge_date, ca.amount
```

## Query 2 — att1 Daily Conversion (Panel 1 table)
Returns: charge_date, att1_total, att1_completions
Used to update: `att1_daily` dict in gen_dashboard.py

```sql
WITH ft_paid AS (
    SELECT DISTINCT paid._id AS plan_id
    FROM alright.premiumplanhistories paid
    INNER JOIN alright.premiumplanhistories ft
        ON paid.userId = ft.userId AND ft.isFreeTrial = 1 AND ft.createdAt >= '2026-02-01'
    WHERE paid.isFreeTrial = 0
),
ranked AS (
    SELECT fp.plan_id,
           ca.chargeAttemptStatus AS status,
           ca.createdAt,
           ROW_NUMBER() OVER (PARTITION BY fp.plan_id ORDER BY ca.createdAt) AS att_num
    FROM alright.chargeattempts ca
    INNER JOIN ft_paid fp ON ca.premiumPlanHistoryId = fp.plan_id
    WHERE ca.chargeAttemptStatus != 'Initiated'
)
SELECT toDate(createdAt) AS charge_date,
       count() AS att1_total,
       countIf(status = 'Completed') AS att1_completions
FROM ranked
WHERE att_num = 1
  AND toDate(createdAt) >= '2026-03-01'
GROUP BY charge_date
ORDER BY charge_date
```

## Query 3 — Cohort Failures by Cohort Date (Panel 3 chart)
Returns: cohort_date, attempted, completed, insuff_funds, abandoned, other_failed, avg_amount
Used to update: `cohort_raw` list in gen_dashboard.py
Insuff funds = failureReason LIKE '%DEBIT%' OR failureReason LIKE '%nsufficien%'

```sql
WITH ft_paid AS (
    SELECT DISTINCT paid._id AS plan_id, paid.userId
    FROM alright.premiumplanhistories paid
    INNER JOIN alright.premiumplanhistories ft
        ON paid.userId = ft.userId AND ft.isFreeTrial = 1 AND ft.createdAt >= '2026-02-01'
    WHERE paid.isFreeTrial = 0
),
plan_cohort AS (
    SELECT fp.plan_id, toDate(ft.createdAt) AS cohort_date
    FROM alright.premiumplanhistories ft
    INNER JOIN ft_paid fp ON ft.userId = fp.userId
    WHERE ft.isFreeTrial = 1 AND ft.createdAt >= '2026-02-01'
    QUALIFY ROW_NUMBER() OVER (PARTITION BY fp.plan_id ORDER BY ft.createdAt DESC) = 1
),
att1 AS (
    SELECT fp.plan_id AS plan_id, pc.cohort_date AS cohort_date,
           ca.amount AS amount, ca.chargeAttemptStatus AS status, ca.failureReason AS failureReason
    FROM alright.chargeattempts ca
    INNER JOIN ft_paid fp ON ca.premiumPlanHistoryId = fp.plan_id
    INNER JOIN plan_cohort pc ON fp.plan_id = pc.plan_id
    WHERE ca.chargeAttemptStatus != 'Initiated'
    QUALIFY ROW_NUMBER() OVER (PARTITION BY fp.plan_id ORDER BY ca.createdAt) = 1
)
SELECT cohort_date,
       count() AS attempted,
       countIf(status = 'Completed') AS completed,
       countIf(status = 'Failed' AND (failureReason LIKE '%DEBIT%' OR failureReason LIKE '%nsufficien%')) AS insuff_funds,
       countIf(status = 'Abandoned') AS abandoned,
       countIf(status = 'Failed' AND NOT (failureReason LIKE '%DEBIT%' OR failureReason LIKE '%nsufficien%')) AS other_failed,
       round(avg(amount)) AS avg_amount
FROM att1
WHERE cohort_date >= '2026-03-01'
GROUP BY cohort_date
ORDER BY cohort_date
```

## Query 4 — Amount by Attempt Performance (Panel 4 cards)
Returns: att_num, amount, resolved, completions, conv_pct, rev_per_user
Used to update: `amt_by_att` list in gen_dashboard.py

```sql
WITH ft_paid AS (
    SELECT DISTINCT paid._id AS plan_id
    FROM alright.premiumplanhistories paid
    INNER JOIN alright.premiumplanhistories ft
        ON paid.userId = ft.userId AND ft.isFreeTrial = 1 AND ft.createdAt >= '2026-02-01'
    WHERE paid.isFreeTrial = 0
),
ranked AS (
    SELECT fp.plan_id AS plan_id, ca.amount AS amount,
           ca.chargeAttemptStatus AS status,
           ROW_NUMBER() OVER (PARTITION BY fp.plan_id ORDER BY ca.createdAt) AS att_num
    FROM alright.chargeattempts ca
    INNER JOIN ft_paid fp ON ca.premiumPlanHistoryId = fp.plan_id
    WHERE ca.chargeAttemptStatus != 'Initiated'
)
SELECT att_num,
       amount,
       count() AS resolved,
       countIf(status = 'Completed') AS completions,
       round(countIf(status = 'Completed') / count() * 100, 2) AS conv_pct,
       round(countIf(status = 'Completed') * amount / count(), 2) AS rev_per_user
FROM ranked
WHERE att_num <= 10
GROUP BY att_num, amount
HAVING resolved >= 200
ORDER BY att_num, amount
```

## Query 5 — Cohort AOV Pivot (Panel 2 — the large table)
Returns: cohort_date, att1_amount, att_num, att_amount, comp_count
Used to update: `pivot_cohort_raw` and the loaded att_data in gen_dashboard.py
Run two sub-queries:

### 5a — Cohort-level stats
```sql
WITH ft_paid AS (
    SELECT DISTINCT paid._id AS plan_id, paid.userId
    FROM alright.premiumplanhistories paid
    INNER JOIN alright.premiumplanhistories ft
        ON paid.userId = ft.userId AND ft.isFreeTrial = 1 AND ft.createdAt >= '2026-02-01'
    WHERE paid.isFreeTrial = 0
),
plan_cohort AS (
    SELECT fp.plan_id, toDate(ft.createdAt) AS cohort_date
    FROM alright.premiumplanhistories ft
    INNER JOIN ft_paid fp ON ft.userId = fp.userId
    WHERE ft.isFreeTrial = 1 AND ft.createdAt >= '2026-02-01'
    QUALIFY ROW_NUMBER() OVER (PARTITION BY fp.plan_id ORDER BY ft.createdAt DESC) = 1
),
ranked AS (
    SELECT fp.plan_id AS plan_id, pc.cohort_date AS cohort_date, ca.amount AS amount,
           ca.chargeAttemptStatus AS status,
           ROW_NUMBER() OVER (PARTITION BY fp.plan_id ORDER BY ca.createdAt) AS att_num
    FROM alright.chargeattempts ca
    INNER JOIN ft_paid fp ON ca.premiumPlanHistoryId = fp.plan_id
    INNER JOIN plan_cohort pc ON fp.plan_id = pc.plan_id
    WHERE ca.chargeAttemptStatus != 'Initiated'
),
att1_amts AS (
    SELECT plan_id, amount AS att1_amount FROM ranked WHERE att_num = 1
)
SELECT r.cohort_date, a.att1_amount,
       COUNT(DISTINCT r.plan_id) AS cohort_size,
       countIf(r.status = 'Completed') AS total_done,
       sumIf(r.amount, r.status = 'Completed') AS total_revenue
FROM ranked r INNER JOIN att1_amts a ON r.plan_id = a.plan_id
GROUP BY r.cohort_date, a.att1_amount
ORDER BY r.cohort_date, a.att1_amount
```

### 5b — Per-attempt breakdown (large result — save to file)
```sql
WITH ft_paid AS (
    SELECT DISTINCT paid._id AS plan_id, paid.userId
    FROM alright.premiumplanhistories paid
    INNER JOIN alright.premiumplanhistories ft
        ON paid.userId = ft.userId AND ft.isFreeTrial = 1 AND ft.createdAt >= '2026-02-01'
    WHERE paid.isFreeTrial = 0
),
plan_cohort AS (
    SELECT fp.plan_id, toDate(ft.createdAt) AS cohort_date
    FROM alright.premiumplanhistories ft
    INNER JOIN ft_paid fp ON ft.userId = fp.userId
    WHERE ft.isFreeTrial = 1 AND ft.createdAt >= '2026-02-01'
    QUALIFY ROW_NUMBER() OVER (PARTITION BY fp.plan_id ORDER BY ft.createdAt DESC) = 1
),
ranked AS (
    SELECT fp.plan_id AS plan_id, pc.cohort_date AS cohort_date, ca.amount AS amount,
           ca.chargeAttemptStatus AS status,
           ROW_NUMBER() OVER (PARTITION BY fp.plan_id ORDER BY ca.createdAt) AS att_num
    FROM alright.chargeattempts ca
    INNER JOIN ft_paid fp ON ca.premiumPlanHistoryId = fp.plan_id
    INNER JOIN plan_cohort pc ON fp.plan_id = pc.plan_id
    WHERE ca.chargeAttemptStatus != 'Initiated'
),
att1_amts AS (
    SELECT plan_id, amount AS att1_amount FROM ranked WHERE att_num = 1
)
SELECT r.cohort_date, a.att1_amount, r.att_num, r.amount AS att_amount,
       countIf(r.status = 'Completed') AS comp_count
FROM ranked r INNER JOIN att1_amts a ON r.plan_id = a.plan_id
GROUP BY r.cohort_date, a.att1_amount, r.att_num, r.amount
ORDER BY r.cohort_date, a.att1_amount, r.att_num, r.amount
```

## After updating data
1. Update the arrays/dicts in gen_dashboard.py
2. For pivot data: save query 5b result as JSON to `pivot_att_data.json` in the site folder,
   and update gen_dashboard.py to read from that file (update `_piv_file` path).
3. Run: `python3 gen_dashboard.py`
4. The script writes `index.html` to the ft-dashboard-site folder (update the `out` path if needed).
5. Git commit and push:
   ```
   cd "ft-dashboard-site"
   git add index.html
   git commit -m "Daily update — {today's date}"
   git push origin master
   ```
